# Day 2 提示词

这些提示词可以用于第二天继续推进这个 AI 任务流管理系统原型。

## 1. 让 AI 检查原型是否完整

```text
请检查这个 AI 任务流管理系统的任务列表页原型。

请判断它是否包含以下内容：
- 任务名称
- 负责人
- 状态
- 创建时间
- 操作按钮
- 状态筛选

请用表格输出：
- 是否包含
- 当前实现方式
- 是否只是演示
- 如果要变成真功能，需要补什么
```

## 2. 区分真功能和演示功能

```text
请帮我检查这个任务列表页原型。

请把页面上的功能分成三类：
1. 当前真的能在前端运行的功能
2. 只是前端演示效果的功能
3. 未来需要后端 API 和数据库支持的功能

请用小白能看懂的话解释，不要太技术化。
```

## 3. 找出所有假数据

```text
请指出这个页面上哪些数据是假数据。

请按页面区域列出来：
- 任务列表
- 负责人
- 状态
- 时间
- 统计数字
- 任务详情
- 执行步骤
- 备注

同时请说明这些假数据未来应该来自哪里。
```

## 4. 规划需要哪些 API

```text
请根据这个 AI 任务流管理系统任务列表页，帮我列出未来需要哪些 API。

请用表格输出：
- 页面功能
- 需要的 API
- API 是用来读取数据还是保存数据
- 是否必须接数据库
- 小白解释
```

## 5. 把原型转成 spec

```text
请把这个任务列表页原型转成一份 spec。

请包含：
- 页面目标
- 用户角色
- 页面模块
- 字段说明
- 操作规则
- 状态规则
- 筛选规则
- 异常情况
- 哪些地方需要 API
- 哪些地方需要数据库

请用清楚、直白、适合开发理解的方式写。
```

## 6. 完善任务列表页

```text
请在原来的任务列表页基础上继续完善。

要求增加：
- 批量选择任务
- 批量标记完成
- 批量归档
- 按负责人筛选
- 按创建时间排序
- 空状态提示
- 加载状态
- 错误状态

仍然先用前端假数据实现，但请明确标注哪些只是演示。
```

## 7. 设计任务详情页

```text
请为 AI 任务流管理系统设计一个任务详情页原型。

页面需要包含：
- 任务基本信息
- 当前状态
- 负责人
- 创建时间
- 更新时间
- 使用的智能体
- 输入内容
- AI 输出结果
- 执行步骤
- 运行日志
- 人工确认区
- 重试按钮
- 标记完成按钮

请先输出页面结构和模块说明，再给最简单可运行版本。
```

## 8. 设计后端数据库字段

```text
请根据这个任务列表页，帮我设计最简单的数据库表。

至少包含：
- tasks 任务表
- users 用户表
- agents 智能体表
- task_steps 任务步骤表
- task_logs 运行日志表

请用小白能看懂的话解释每张表是干什么的。
```

## 9. 给老板演示时怎么说

```text
我要给老板演示这个 AI 任务流管理系统原型。

请帮我准备一段 2 分钟的讲解稿。

要求：
- 说清楚这个页面解决什么问题
- 说清楚现在哪些是真功能
- 说清楚哪些是假数据
- 说清楚下一步要做什么
- 语言通俗，不要太技术化
```

## 10. 判断是否可以上线

```text
请帮我判断这个原型距离上线还差什么。

请分成：
- 现在已有
- 还缺前端
- 还缺后端
- 还缺数据库
- 还缺权限
- 还缺测试
- 上线前必须完成

请用非常直白的话解释。
```

## 最重要的提醒

每次继续做原型时，都要问 AI：

```text
请明确告诉我：
- 哪些功能是真的
- 哪些功能只是演示
- 哪些数据是假数据
- 哪些地方未来需要 API
- 哪些地方未来需要数据库
```

这样可以避免把“看起来能用”误以为“真的能用”。
