# AI 任务流管理系统：任务列表页原型说明

## 这个页面是做什么的

这个页面是一个“AI 任务流管理系统”的任务列表页原型。

它的作用是让用户快速看到：

- 有哪些 AI 任务
- 每个任务由谁负责
- 当前任务是什么状态
- 任务什么时候创建
- 任务最近什么时候更新
- 哪些任务需要人工确认
- 哪些任务异常了
- 用户可以对任务做哪些操作

简单说：

> 这是一个用来管理 AI 任务执行情况的工作台页面。

## 页面包含的主要模块

### 1. 左侧导航

用于切换系统里的不同功能区。

当前原型里包含：

- 任务列表
- 智能体
- 知识库
- 运行日志
- 设置

这些目前主要是页面结构展示，还没有真的跳转功能。

### 2. 顶部标题区

显示当前页面名称：

- 任务列表

并提供一个按钮：

- 新建任务

目前“新建任务”按钮只是演示效果，点击后弹出提示，还不会真的创建任务。

### 3. 任务概览区

页面顶部有 4 个统计数字：

- 进行中
- 待人工确认
- 今日完成
- 异常任务

这些数字现在是根据前端假数据临时算出来的，不是真实业务统计。

### 4. 筛选工具栏

包含：

- 搜索框
- 状态筛选
- 优先级筛选

用户可以用它快速找到任务。

目前筛选只是在页面里的假数据中筛选，还没有调用后端 API。

### 5. 任务列表

任务列表现在已经包含：

- 任务名称
- 负责人
- 状态
- 创建时间
- 更新时间
- 优先级
- 智能体
- 操作按钮

操作按钮包括：

- 查看
- 编辑
- 归档
- 重试

其中“重试”只在异常任务上显示。

### 6. 右侧任务详情

点击某个任务后，右侧会显示：

- 任务标题
- 任务描述
- 当前进度
- 备注
- 标记待确认
- 标记完成

这里可以帮助用户快速了解某个任务现在进行到哪一步。

## 当前哪些是真的

当前原型里，以下内容在页面当前运行时是真的能操作：

- 搜索任务
- 按状态筛选
- 按优先级筛选
- 点击任务查看详情
- 标记任务为待确认
- 标记任务为完成
- 点击行内操作按钮弹出提示

但这些操作只在当前浏览器页面里生效。

刷新页面后，改动会消失。

## 当前哪些只是演示

以下部分只是前端演示：

- 任务数据
- 负责人
- 智能体
- 创建时间
- 更新时间
- 任务状态
- 执行步骤
- 备注
- 新建任务
- 编辑任务
- 归档任务
- 重试任务
- 顶部统计数字

它们现在没有连接真实数据库，也没有后端 API。

## 未来要变成真系统，需要补什么

未来如果要上线，需要增加：

- 后端 API
- 数据库
- 用户登录
- 权限控制
- 真实任务创建
- 真实任务状态更新
- AI 任务执行记录
- 运行日志
- 异常重试机制
- 数据保存
- 错误提示

一句话：

> 当前原型是“能看、能点、能讨论”的版本；未来真系统要做到“能保存、能执行、能长期使用”。
