# 任务列表页前端页面 Spec

## 1. 页面目标

本页面用于集中展示和管理 AI 任务流中的任务，帮助业务负责人、运营人员和任务负责人快速了解任务执行情况，并对任务进行筛选、查看和处理。

页面需要支持用户完成以下目标：

- 查看当前所有 AI 任务
- 快速判断进行中、待确认、已完成和异常任务数量
- 按任务名称、负责人、智能体搜索任务
- 按状态和优先级筛选任务
- 点击任务查看详情
- 对任务执行查看、编辑、归档、重试、标记待确认、标记完成等操作

## 2. 页面模块

### 2.1 页面头部

展示页面名称和主要入口操作。

内容：

- 页面标题：AI 任务流管理
- 页面说明：集中查看和处理 AI Agent 执行任务
- 新建任务按钮

### 2.2 任务概览统计

展示当前任务整体情况。

统计项：

- 进行中数量
- 待人工确认数量
- 今日完成数量
- 异常任务数量

要求：

- 统计数字基于当前任务数据计算
- 任务状态变化后，统计数字同步更新
- 异常任务需要视觉强调

### 2.3 搜索和筛选区

用于缩小任务列表范围。

包含：

- 搜索框
- 状态筛选
- 优先级筛选
- 重置筛选按钮

### 2.4 任务列表区

展示任务基础信息，是页面核心区域。

建议形态：

- 桌面端使用表格
- 如果采用卡片，也必须完整展示基础字段

### 2.5 任务详情区

用户点击任务后，在右侧展示该任务详情。

展示内容：

- 任务标题
- 任务描述
- 当前执行步骤
- 备注
- 标记待确认按钮
- 标记完成按钮

### 2.6 任务操作区

每条任务提供操作按钮。

普通任务：

- 查看
- 编辑
- 归档

异常任务：

- 查看
- 编辑
- 重试

## 3. 字段列表

### 3.1 任务列表字段

| 字段 | 说明 | 是否必填 | 展示要求 |
| --- | --- | --- | --- |
| 任务名称 | AI 任务的标题 | 是 | 清晰可读，支持点击查看详情 |
| 任务描述 | 任务简要说明 | 是 | 可展示摘要，过长时省略 |
| 状态 | 当前任务状态 | 是 | 使用明显状态标签 |
| 优先级 | 任务优先级 | 是 | 高优先级需要强调 |
| 负责人 | 当前任务负责人 | 是 | 支持被搜索 |
| 智能体 | 执行任务的 AI Agent | 是 | 支持被搜索 |
| 创建时间 | 任务创建时间 | 是 | 格式统一 |
| 更新时间 | 最近更新时间 | 是 | 格式统一 |
| 操作 | 当前可执行操作 | 是 | 根据任务状态展示不同按钮 |

### 3.2 任务状态

| 状态 | 说明 | 展示建议 |
| --- | --- | --- |
| 进行中 | AI 任务正在执行 | 蓝色标签 |
| 待确认 | 需要人工确认结果 | 黄色或橙色标签 |
| 已完成 | 任务已完成 | 绿色标签 |
| 异常 | 任务执行失败或出错 | 红色标签，明显强调 |

### 3.3 任务优先级

| 优先级 | 说明 | 展示建议 |
| --- | --- | --- |
| 高 | 需要优先处理 | 强调样式 |
| 中 | 正常优先级 | 普通样式 |
| 低 | 可稍后处理 | 弱化样式 |

### 3.4 任务详情字段

| 字段 | 说明 |
| --- | --- |
| 任务标题 | 当前任务名称 |
| 任务描述 | 任务完整描述 |
| 当前状态 | 当前任务状态 |
| 负责人 | 当前任务负责人 |
| 智能体 | 执行任务的 AI Agent |
| 优先级 | 当前任务优先级 |
| 执行步骤 | AI 任务执行过程 |
| 备注 | 人工补充说明 |

## 4. 筛选项

### 4.1 搜索

搜索框支持以下字段：

- 任务名称
- 负责人
- 智能体

搜索规则：

- 输入关键词后，任务列表展示匹配结果
- 搜索与状态筛选、优先级筛选同时生效
- 无匹配结果时展示空状态

### 4.2 状态筛选

选项：

- 全部状态
- 进行中
- 待确认
- 已完成
- 异常

规则：

- 选择某个状态后，只展示对应状态的任务
- 默认选中全部状态

### 4.3 优先级筛选

选项：

- 全部优先级
- 高优先级
- 中优先级
- 低优先级

规则：

- 选择某个优先级后，只展示对应优先级的任务
- 默认选中全部优先级

### 4.4 重置筛选

点击后：

- 清空搜索关键词
- 状态恢复为全部状态
- 优先级恢复为全部优先级
- 列表恢复展示默认任务范围

## 5. 操作按钮

### 5.1 页面级按钮

| 按钮 | 位置 | 行为 |
| --- | --- | --- |
| 新建任务 | 页面头部 | 打开新建任务表单或跳转新建任务页 |
| 重置筛选 | 搜索筛选区 | 清空搜索和筛选条件 |

### 5.2 行级按钮

| 按钮 | 适用任务 | 行为 |
| --- | --- | --- |
| 查看 | 全部任务 | 展示该任务详情 |
| 编辑 | 全部任务 | 打开编辑任务表单 |
| 归档 | 非异常任务 | 将任务从默认列表移出或标记为已归档 |
| 重试 | 异常任务 | 重新触发异常任务执行 |

### 5.3 详情区按钮

| 按钮 | 行为 |
| --- | --- |
| 标记待确认 | 将当前任务状态更新为待确认 |
| 标记完成 | 将当前任务状态更新为已完成 |

## 6. 交互规则

### 6.1 页面加载

- 用户进入页面后，请求任务列表数据
- 加载中展示骨架屏或 loading
- 数据加载成功后展示统计区、筛选区和任务列表
- 默认选中第一条任务，或详情区展示未选择任务提示

### 6.2 搜索和筛选

- 搜索关键词变化后，列表实时更新
- 状态筛选变化后，列表实时更新
- 优先级筛选变化后，列表实时更新
- 搜索、状态筛选、优先级筛选必须同时生效
- 筛选结果变化后，统计区可保持全量统计，也可展示筛选后统计，但同一版本内规则必须一致

### 6.3 查看详情

- 点击任务行后，右侧详情区展示对应任务
- 切换不同任务时，详情内容同步切换
- 当前选中的任务行需要有高亮状态
- 点击行内操作按钮时，不应触发行点击逻辑

### 6.4 编辑任务

- 点击编辑后，打开编辑任务表单
- 表单应回填当前任务基础信息
- 保存成功后，列表和详情同步更新
- 保存失败时展示错误提示，原数据不应丢失

### 6.5 归档任务

- 点击归档后，应出现确认提示
- 用户确认后再执行归档
- 归档成功后，任务从默认列表中移出或展示为已归档
- 归档失败时保留任务原状态

### 6.6 重试异常任务

- 只有异常任务展示重试按钮
- 点击重试后，任务进入重新执行流程
- 原型阶段可弹出演示提示
- 正式系统中应调用后端 API
- 重试失败时展示错误提示

### 6.7 状态更新

- 点击标记待确认后，当前任务状态变为待确认
- 点击标记完成后，当前任务状态变为已完成
- 状态更新后，列表状态标签同步更新
- 状态更新后，顶部统计数字同步更新
- 更新失败时，任务状态回到原状态

### 6.8 新建任务

- 点击新建任务后，打开新建任务表单
- 表单字段建议包括任务名称、任务描述、负责人、智能体、优先级、输入内容
- 新建成功后，新任务出现在任务列表中
- 正式系统中刷新页面后新任务仍然存在

## 7. 空状态

### 7.1 无任何任务

触发条件：

- 系统中没有任务数据

展示文案：

- 标题：暂无 AI 任务
- 说明：创建第一个任务，开始管理 AI Agent 工作流。
- 按钮：新建任务

### 7.2 搜索或筛选无结果

触发条件：

- 有任务数据，但当前搜索和筛选条件下没有匹配结果

展示文案：

- 标题：没有找到匹配的任务
- 说明：尝试修改搜索关键词或筛选条件。
- 按钮：清空筛选

### 7.3 未选择任务

触发条件：

- 右侧详情区没有当前选中任务

展示文案：

- 标题：请选择一个任务查看详情
- 说明：点击左侧任务列表中的任意任务，可查看执行步骤和备注。

## 8. 错误状态

### 8.1 任务列表加载失败

展示内容：

- 标题：任务加载失败
- 说明：请检查网络连接后重试。
- 按钮：重新加载

交互：

- 点击重新加载后，重新请求任务列表

### 8.2 任务详情加载失败

展示内容：

- 标题：任务详情加载失败
- 说明：当前任务详情暂时无法展示。
- 按钮：重新加载详情

### 8.3 操作失败

适用场景：

- 编辑失败
- 归档失败
- 重试失败
- 状态更新失败
- 新建任务失败

提示方式：

- toast 或 message

文案示例：

- 保存失败，请稍后重试
- 归档失败，请稍后重试
- 重试任务失败，请稍后重试
- 状态更新失败，请稍后重试

规则：

- 操作失败后，不应错误改变任务状态
- 当前搜索和筛选条件需要保留
- 当前选中任务需要保留
- 列表不应整块消失

## 9. 验收标准

### 9.1 页面字段验收

- 页面包含任务名称、负责人、状态、创建时间、更新时间和操作按钮
- 页面包含状态筛选
- 页面包含优先级筛选
- 页面包含搜索框
- 页面包含任务概览统计

### 9.2 列表展示验收

- 用户进入页面后，可以看到任务列表
- 每条任务展示完整基础字段
- 不同状态有明显区分
- 异常任务可以被快速识别
- 高优先级任务有明显展示

### 9.3 搜索和筛选验收

- 输入任务名称关键词，可以筛选出匹配任务
- 输入负责人关键词，可以筛选出匹配任务
- 输入智能体关键词，可以筛选出匹配任务
- 选择状态后，只展示对应状态任务
- 选择优先级后，只展示对应优先级任务
- 搜索、状态筛选、优先级筛选可以同时生效
- 无匹配结果时，页面显示空状态提示
- 点击重置筛选后，恢复默认列表

### 9.4 详情查看验收

- 点击任务后，右侧显示对应任务详情
- 切换不同任务时，详情内容同步变化
- 详情中可以看到任务描述、执行步骤和备注
- 当前选中任务在列表中有高亮状态

### 9.5 操作按钮验收

- 每条任务都有操作按钮
- 普通任务显示查看、编辑、归档
- 异常任务显示查看、编辑、重试
- 点击操作按钮时，不应误触发行点击逻辑
- 原型阶段，按钮可以先弹出演示提示

### 9.6 状态更新验收

- 点击标记待确认后，当前任务状态变为待确认
- 点击标记完成后，当前任务状态变为已完成
- 顶部统计数字随状态变化同步更新
- 状态更新失败时，任务状态不应被错误修改

### 9.7 加载、空状态和错误状态验收

- 首次加载任务时，有 loading 或骨架屏
- 没有任何任务时，展示无任务空状态
- 搜索或筛选无结果时，展示无匹配结果空状态
- 任务列表加载失败时，展示错误提示和重新加载按钮
- 操作失败时，有明确错误提示

### 9.8 正式系统验收

- 任务数据来自后端 API
- 新建、编辑、状态更新、归档、重试等操作调用后端 API
- 数据保存到数据库
- 刷新页面后数据不丢失
- 接口失败时有错误提示
- 用户只能看到自己有权限查看的任务
