# Codex 常用 Prompt 模板

## 1. 学概念

```text
我是小白，请用通俗直白的话解释【概念】。
不要用太多专业术语，最好用一个生活中的例子说明。
```

示例：

```text
我是小白，请用通俗直白的话解释 HTML 文件。
不要用太多专业术语，最好用一个生活中的例子说明。
```

## 2. 对比两个概念

```text
请用小白能听懂的话解释【概念 A】和【概念 B】的区别。
请用表格总结，并举一个任务管理系统的例子。
```

示例：

```text
请用小白能听懂的话解释 Codex 和 ChatGPT 问答的区别。
请用表格总结，并举一个任务管理系统的例子。
```

## 3. 让 Codex 先不要写代码

```text
我想做一个【产品类型】。
请先不要写代码。
请先帮我整理：
1. 需求应该怎么描述
2. 任务应该怎么拆
3. 每一步应该怎么验收
```

示例：

```text
我想用 Codex 学会构建一个小型产品。
请先不要写代码。
请告诉我我应该如何描述需求、如何拆任务、如何验收结果。
```

## 4. 描述一个产品需求

```text
我想做一个【产品名称】，给【目标用户】使用。
它主要解决【用户问题】。
第一版需要这些功能：
- 【功能 1】
- 【功能 2】
- 【功能 3】

暂时不需要：
- 【不做功能 1】
- 【不做功能 2】

界面风格希望【风格描述】。
验收标准是【做到什么算完成】。
请先帮我整理 MVP 功能范围，不要写代码。
```

示例：

```text
我想做一个 AI 任务流管理系统，给个人用户使用。
它主要解决任务太多、不知道先做什么的问题。
第一版需要这些功能：
- 新建任务
- 查看任务列表
- 标记完成
- 删除任务

暂时不需要：
- 团队协作
- 支付
- 手机 App

界面风格希望简洁清爽。
验收标准是用户可以管理自己的任务，刷新后数据不丢失。
请先帮我整理 MVP 功能范围，不要写代码。
```

## 5. 生成首页原型

```text
创建一个学习项目文件夹，生成一个最简单的产品首页原型。
页面内容包含：
- 产品名称：【产品名称】
- 一句话说明：【一句话说明】
- 三个核心功能：【功能 1】、【功能 2】、【功能 3】
- 一个【按钮名称】按钮
```

示例：

```text
创建一个学习项目文件夹，生成一个最简单的产品首页原型。
页面内容包含：
- 产品名称：AI 任务流管理系统
- 一句话说明
- 三个核心功能
- 一个进入任务列表的按钮
```

## 6. 修改页面风格

```text
请修改当前首页原型，让它看起来更【风格】。
要求：
- 保留原有内容
- 页面更清晰
- 按钮更明显
- 手机和电脑都能正常查看
```

示例：

```text
请修改当前首页原型，让它看起来更像一个清爽的效率工具。
要求：
- 保留原有内容
- 页面更清晰
- 按钮更明显
- 手机和电脑都能正常查看
```

## 7. 解释已有页面

```text
请解释这个 HTML 页面是怎么组成的。
请用小白能听懂的话说明：
1. 哪些是页面内容
2. 哪些是页面样式
3. 每个页面模块分别是什么
```

## 8. 拆页面模块

```text
请把这个产品首页拆成页面模块。
每个模块请说明：
1. 模块名称
2. 模块作用
3. 用户看到什么
4. 后续可以怎么扩展
```

## 9. 设计任务列表页

```text
请先不要写代码。
请帮我设计 AI 任务流管理系统的任务列表页。
请说明：
1. 页面上应该有哪些模块
2. 用户可以做哪些操作
3. 哪些地方先用假数据
4. 以后哪些地方要接真实数据
```

## 10. 从假数据过渡到真实数据

```text
请用小白能听懂的话说明：
当前页面里的哪些内容是假数据？
如果以后要变成真实数据，需要增加哪些部分？
请用任务管理系统举例。
```

## 11. 让 Codex 实现一个小功能

```text
请在当前项目中实现【功能名称】。
要求：
1. 先阅读相关文件
2. 按现有代码风格修改
3. 不改无关内容
4. 完成后告诉我修改了哪些文件
5. 告诉我如何手动验收
```

示例：

```text
请在当前项目中实现任务列表页面。
要求：
1. 先阅读相关文件
2. 按现有代码风格修改
3. 不改无关内容
4. 完成后告诉我修改了哪些文件
5. 告诉我如何手动验收
```

## 12. 验收结果

```text
请帮我检查当前结果是否满足需求。
请按以下格式回答：
1. 已完成什么
2. 是否符合需求
3. 如何手动验收
4. 还有哪些问题或下一步建议
```

## 13. 学习复盘

```text
请根据我们刚才完成的内容，帮我做一次小白复盘。
请说明：
1. 今天学了哪些概念
2. 每个概念是什么意思
3. 我们实际创建了什么文件
4. 下一步适合学什么
```
