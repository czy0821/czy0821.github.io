# Codex 产品学习笔记

## 1. Codex、Prompt、Context

### Codex

Codex 是一个能帮你做开发工作的 AI 助手。

它可以：

- 看项目文件
- 解释代码和概念
- 创建或修改文件
- 生成页面原型
- 运行命令和测试
- 帮你检查结果

简单理解：

> Codex 像一个会写代码、会看项目、会陪你做产品的小助教。

### Prompt

Prompt 是你给 AI 的指令或问题。

比如：

```text
请用零基础能听懂的语言解释前端是什么。
```

或者：

```text
创建一个学习项目文件夹，生成一个最简单的产品首页原型。
页面内容包含产品名称、一句话说明、三个核心功能和一个按钮。
```

简单理解：

> Prompt = 你告诉 AI 要做什么。

Prompt 越清楚，AI 越容易做对。

### Context

Context 是上下文，也就是背景信息。

比如：

```text
我是小白。
我正在学习用 Codex 做一个小型产品。
我已经有一个 AI 任务流管理系统的首页原型。
```

简单理解：

> Context = 让 AI 理解你现在的情况。

## 2. 前端、后端、数据库、API、部署

### 前端

前端是用户能看到、能点击的部分。

比如：

- 页面
- 按钮
- 表单
- 任务列表
- 登录界面

### 后端

后端是用户看不见、但负责处理事情的部分。

比如：

- 检查用户是否登录
- 保存任务
- 删除任务
- 判断数据是否正确

### 数据库

数据库是保存数据的地方。

比如保存：

- 用户账号
- 任务标题
- 任务状态
- 创建时间

### API

API 是前端和后端之间沟通的接口。

比如前端说：

```text
我要新建一个任务。
```

后端通过 API 接收请求，处理后再告诉前端：

```text
任务创建成功。
```

### 部署

部署就是把项目放到网上，让别人可以通过网址访问。

本地能打开，不代表别人能打开；部署之后，产品才真正上线。

## 3. HTML 文件和本地运行

### HTML 文件

HTML 文件就是网页文件。

它告诉浏览器：

- 页面上显示什么标题
- 显示什么文字
- 有哪些按钮
- 有哪些链接
- 页面大概有哪些结构

简单理解：

> HTML 文件就是装网页内容的文件。

一个 HTML 文件可以算是一个最简单的网站页面。

### 本地运行

本地运行就是把项目在自己的电脑上打开或启动起来。

比如简单 HTML 页面：

```text
直接用浏览器打开 index.html
```

如果是更完整的前端项目，可能会用：

```bash
npm run dev
```

然后在浏览器访问：

```text
http://localhost:3000
```

其中 `localhost` 表示自己的这台电脑。

## 4. 产品项目通常由哪些部分组成

一个产品项目通常包括：

- 产品需求
- 页面和交互设计
- 前端
- 后端
- 数据库
- API
- 测试
- 部署
- 运营和反馈

可以简单记成：

```text
需求 -> 设计 -> 前端 -> 后端 -> 数据库 -> API -> 测试 -> 部署 -> 反馈迭代
```

对于小白学习项目，最开始可以只关注：

```text
需求清单 + 首页原型 + 一个可用功能 + 数据保存 + 简单部署
```

## 5. Vibe coding、原型、页面模块、交互、假数据、真实数据

### Vibe coding

Vibe coding 是用自然语言和 AI 边试边做。

比如：

```text
把首页做得更清爽一点。
按钮更明显一点。
整体像一个效率工具。
```

适合：

- 快速做原型
- 学习编程
- 把想法变成可看的页面
- 边做边调整

### 原型

原型是产品最早的样子，用来先看方向对不对。

它不一定完整可用。

比如一个首页原型可以只有：

- 产品名称
- 一句话说明
- 核心功能
- 一个按钮

### 页面模块

页面模块是页面里一块一块的区域。

比如：

- 顶部导航
- 产品介绍
- 功能展示
- 按钮区域
- 底部信息

### 交互

交互是用户操作后，页面给出的反应。

比如：

- 点击按钮跳转
- 鼠标移到按钮上变色
- 输入内容后显示提示
- 点击完成后任务状态改变

### 假数据

假数据是为了展示效果，临时写在页面里的示例数据。

比如：

```text
整理产品需求
设计首页原型
准备明天计划
```

这些内容只是展示用，不一定真的来自数据库。

### 真实数据

真实数据是用户真正创建、保存、修改的数据。

比如用户真的添加一条任务：

```text
今晚 8 点学习 Codex
```

刷新页面后它还在，这就是一条真实数据。

## 6. PRD 和用户故事

### PRD

PRD 是产品需求文档，用来写清楚这个产品到底要做什么。

通常包括：

- 产品目标
- 目标用户
- 核心功能
- 不做范围
- 页面说明
- 用户流程
- 验收标准

### 用户故事

用户故事是用用户视角描述需求。

常见格式：

```text
作为一个【用户类型】，
我想要【做某件事】，
以便【得到某个好处】。
```

例子：

```text
作为一个个人用户，
我想要新建任务，
以便记录今天要做的事情。
```
